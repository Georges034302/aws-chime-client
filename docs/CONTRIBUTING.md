# Contributing Guide

Thank you for considering contributing to **AWS Chime Client**!  
We welcome improvements, ideas, bug reports, and feature suggestions.

## How to Contribute
1. **Fork** the repository  
2. **Create a new branch** for your feature or fix  
3. **Make your changes** (keep commits clear and concise)  
4. **Open a Pull Request** with a descriptive title and summary  
5. A maintainer will review your PR and collaborate with you

## Coding Standards
- Keep the UI clean and lightweight  
- Avoid adding heavy dependencies  
- Maintain browser compatibility  
- Follow minimalistic UI/UX patterns  
- Use clear and consistent naming  
- Use Amazon Chime SDK v3 API (not v2)  
- Backend uses AWS SDK v3 modular packages  
- Lambda functions use Node.js 18.x runtime  

## Reporting Issues
If you encounter bugs or unexpected behaviors:
- Open an issue with steps to reproduce  
- Include screenshots when possible  
- Suggest enhancements if applicable  

## Thank You
Every contribution helps make this project better.  
We appreciate your effort and collaboration!
